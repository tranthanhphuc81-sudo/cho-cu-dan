Phần 7: Kỹ Thuật - Đăng Nhập và Bảo Mật (Authentication)

Tại sao phần này quan trọng nhất?

Đây là phần quan trọng nhất để bảo vệ Ví tiền của người dùng. Một lỗ hổng bảo mật có thể làm sụp đổ toàn bộ niềm tin của cộng đồng.

Hệ thống xác thực đa lớp
1. Đăng ký tài khoản:
  • SMS OTP: Xác thực số điện thoại thật
  • Email verification: Gửi link kích hoạt
  • Thông tin bắt buộc: Họ tên, Số căn hộ, Tòa nhà
2. Đăng nhập hàng ngày:
  • JWT Token: Thời hạn 24h, tự động gia hạn
  • Refresh Token: Lưu an toàn, thời hạn 30 ngày
  • Device fingerprinting: Nhận dạng thiết bị lạ
Bảo mật giao dịch ví tiền
1. Mã PIN giao dịch:
  • 6 số PIN: Riêng biệt với mật khẩu đăng nhập
  • Khóa tài khoản: Sau 3 lần nhập sai
  • Xác thực sinh trắc học: Face ID/Vân tay (nếu có)
2. Giới hạn giao dịch:
  • Hạn mức ngày: 2.000.000đ
  • Xác thực bổ sung: > 500.000đ cần SMS OTP
  • Lịch sử giao dịch: Lưu đầy đủ, không thể xóa

Cấu trúc Code mẫu (Backend API)

Node.js + Express.js + JWT 

// file: /routes/auth.js
const express = require('express');
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');
const User = require('../models/User');

router.post('/login', async (req, res) => {
    try {
        const { phone, password } = req.body;
        
        // 1. Kiểm tra user tồn tại
        const user = await User.findOne({ phone });
        if (!user) {
            return res.status(401).json({ 
                success: false, 
                message: 'Số điện thoại chưa được đăng ký' 
            });
        }
        
        // 2. Kiểm tra mật khẩu
        const isValidPassword = await bcrypt.compare(password, user.password);
        if (!isValidPassword) {
            return res.status(401).json({ 
                success: false, 
                message: 'Mật khẩu không đúng' 
            });
        }
        
        // 3. Tạo JWT token
        const accessToken = jwt.sign(
            { 
                userId: user._id, 
                phone: user.phone,
                buildingId: user.buildingId 
            }, 
            process.env.JWT_SECRET, 
            { expiresIn: '24h' }
        );
        
        // 4. Tạo Refresh Token
        const refreshToken = jwt.sign(
            { userId: user._id }, 
            process.env.REFRESH_SECRET, 
            { expiresIn: '30d' }
        );
        
        res.json({
            success: true,
            data: {
                accessToken,
                refreshToken,
                user: {
                    id: user._id,
                    name: user.name,
                    phone: user.phone,
                    apartment: user.apartment,
                    buildingName: user.buildingName
                }
            }
        });
        
    } catch (error) {
        res.status(500).json({ 
            success: false, 
            message: 'Lỗi server', 
            error: error.message 
        });
    }
});

Điểm mạnh của dự án
  • Thị trường ngách rõ ràng: Cư dân chung cư có nhu cầu thực tế
